# PRD v2.1 One Page — 확정보드

업무 회의 가용시간을 회의 결정 상태로 바꾸는 도구

## 1. 문제정의

이 과제의 문제는 여러 사람의 빈 시간이 가장 많이 겹치는 구간을 찾는 일이 아니다.

업무 회의에서는 시간이 겹친다는 사실만으로 회의를 확정할 수 없다. 주최자는 응답을 받은 뒤에도 다음을 판단해야 한다.

- 꼭 참석해야 하는 사람이 가능한가
- 최소 몇 명 이상이면 회의를 진행할 수 있는가
- 누가 현재 일정 그대로 가능한가
- 누가 자신의 일정을 조정해야 가능한가
- 미응답자 중 반드시 기다려야 하는 사람이 누구인가
- 지금 정해도 되는가, 아니면 특정 사람에게 한 번 더 확인해야 하는가
- 현재 범위로 어렵다면 무엇을 다시 물어봐야 하는가

따라서 핵심 문제는 `시간 수집`이 아니라 `해석`이다.

서로 다른 가용 시간, 일정 조정 가능성, 선호, 미응답 상태를 회의 성립 여부와 주최자의 다음 행동으로 바꾸는 것이 이 제품이 풀 문제다.

## 2. 솔루션

확정보드는 주최자와 참석자가 알고 있는 시간 정보를 가볍게 수집하고, 제품이 그 정보를 회의 결정 상태로 번역한다.

전체 흐름은 다음과 같다.

```text
회의 기준 설정
→ 조율 가능 시간대 수집
→ 참석자 가용시간 입력
→ 제품이 후보 시간 계산
→ 후보별 결정 상태 해석
→ 주최자의 확정 또는 추가 확인
```

주최자는 먼저 회의 기준을 정한다.

- 회의 목적
- 참석자
- 꼭 참석해야 하는 사람
- 최소 참석 인원
- 회의 가능 기간
- 확보할 시간
- 자신이 회의를 진행할 수 있는 조율 가능 시간대

참석자는 후보 시간을 하나씩 투표하지 않는다. 주최자가 연 조율 가능 시간대 안에서 30분 단위로 자신의 상태를 표시한다.

- `가능해요`: 별도 조정 없이 비울 수 있다
- `내 일정 조정`: 앞뒤 일정이나 개인 계획을 조정하면 비울 수 있다
- `어려워요`: 해당 시간을 비울 수 없다

제품은 이 가용시간을 바탕으로 실제 후보 시간을 계산한다. 예를 들어 주최자가 `09:00-12:00`을 열고 확보 시간이 1시간이면, 제품은 `09:00`, `09:30`, `10:00`, `10:30`, `11:00` 시작 후보를 파생한다.

각 후보는 다음 네 가지 결정 상태로 번역된다.

| 상태                            | 의미                                                       | 주최자의 다음 행동              |
| ------------------------------- | ---------------------------------------------------------- | ------------------------------- |
| 바로 정할 수 있어요             | 필수 참석자와 최소 인원 기준을 충족하고 조정 리스크가 낮다 | 이 시간으로 정하기              |
| 정하기 전에 한 번 확인해 주세요 | 회의는 가능하지만 일정 조정이 필요한 사람이 있다           | 필요한 사람에게 확인            |
| 아직 정하기 어려워요            | 회의 성립에 중요한 응답이 부족하다                         | 필요한 사람에게 응답 요청       |
| 이 시간은 어려워요              | 필수 참석자 불가 또는 최소 인원 미달로 성립하기 어렵다     | 다른 후보 검토 또는 시간대 확장 |

확정 가능한 시간이 없더라도 새 요청을 처음부터 만들게 하지 않는다.

기존 링크와 응답을 유지한 채, 필요한 사람에게만 응답을 요청하거나 주최자의 조율 가능 시간대를 넓힌다. 새로 열린 시간대만 미응답 상태로 열어 같은 조율 안에서 다시 판단한다.

## 3. 설계 이유

### 응답은 가볍게

참석자에게 전체 캘린더 연동, 상세 사유 입력, 선호 순위 입력을 요구하지 않는다. 응답 부담이 커지면 필수 참석자의 응답이 늦어지고, 주최자는 회의 성립 여부를 판단할 수 없다.

다만 단순 가능/불가만으로 줄이지도 않는다. 현재 일정 그대로 가능한 사람과 일정을 바꿔야 가능한 사람은 확정 리스크가 다르기 때문이다.

그래서 기본 응답은 세 단계로 제한한다.

### 해석은 제품이

주최자가 사람별 응답표를 보고 필수 참석자, 최소 인원, 미응답, 일정 조정 여부를 직접 계산하게 하지 않는다.

제품은 가용시간을 실제 후보로 계산하고, 각 후보를 회의 성립 여부와 리스크로 해석한다. 주최자는 원본 데이터를 다시 계산하는 대신 상태, 이유, 다음 행동을 먼저 본다.

요약은 빠른 결정을 돕고, 상세 근거는 제품 판단을 검토하게 한다.

### 결정은 주최자가

제품은 자동으로 회의 시간을 확정하지 않는다.

업무 회의에는 제품이 알 수 없는 맥락이 있다. 회의 중요도, 참석자의 실제 역할, 외근 일정의 변경 가능성, 회의를 미뤘을 때의 업무 영향은 규칙만으로 완전히 판단하기 어렵다.

따라서 제품은 “지금 정할 수 있는지”와 “무엇을 더 확인해야 하는지”를 설명하고, 최종 결정은 주최자에게 남긴다.

### 재조율은 새 시작이 아니라 이어가기

한 번에 확정되지 않는 것은 실패가 아니라 정상적인 회의 조율 과정이다.

중요한 것은 전원에게 다시 묻지 않는 것이다. 제품은 기존 링크와 응답을 유지하고, 회의 성립 여부를 바꿀 사람과 시간대만 다시 확인하게 한다.

이 제품의 핵심은 시간표를 더 예쁘게 보여주는 것이 아니다.

각자의 가용시간을 회의 결정 가능성과 주최자의 다음 행동으로 바꾸는 것이다.
