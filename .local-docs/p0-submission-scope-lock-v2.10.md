# P0 제출 범위 잠금 v2.10

- 상태: 현재 최우선 실행 기준
- 기준일: 2026-07-12
- 상위 문제 정의: `research-internalized-thinking-v1.md`
- 제품 계약: `prd-v2.2.1.md`, `decision-model-v2.2.md`

## 1. 완성 범위

> 회의 기준 설정 → 참석자 응답 → 주최자 판단 → 한 사람에게 재요청 → 응답 변경 후 재판정 → 확정

이 흐름 밖의 기능은 실제 서비스에 필요하더라도 이번 채용 과제의 가설을 직접 검증하지 않는다.

## 2. 이번 과제가 증명할 것

평가자는 별도 설명 없이 후보 하나를 보고 다음을 설명할 수 있어야 한다.

1. 지금 확정할 수 있는가?
2. 왜 가능한가 또는 불가능한가?
3. 반드시 응답해야 하는 특정인은 누구인가?
4. 대체 가능한 참석자 중 몇 명의 가능 응답이 더 필요한가?
5. 한 사람의 응답 변경 뒤 왜 READY가 되었는가?

## 3. P0 필수 화면

### 최소 생성

- 회의 제목과 한 문장 목적
- 주최자를 포함한 대표 fixture 6명
- 필수·선택 참석자와 최소 인원
- 다음 주 날짜 범위와 1시간 기본값
- 기본 조율 범위 확인과 필요한 예외 편집

6명, 다음 주, 1시간은 제품 제약이 아니라 시연 fixture와 기본값이다.

### 참석자 응답

- 더미 캘린더로 시간표 초안 채우기
- 가능·일정 조정·어려움 수정
- 제출 전 초안은 판정에서 미응답 유지
- 제출 뒤 같은 회의 fixture에 반영

### 주최자 결과

- READY·PENDING·IMPOSSIBLE을 같은 보드에서 비교
- 상태의 가장 중요한 이유
- 필수 미응답자
- 대체 가능한 미응답자와 필요한 가능 응답 수
- 상태에 맞는 하나의 우선 행동

### 재판정과 확정

- PENDING 후보와 요청 대상을 주최자가 선택
- DEV 시연 전환으로 해당 참석자 응답 화면을 연다.
- 응답 제출 뒤 동일 회의와 동일 선택 후보를 보존한 결과 보드로 돌아간다.
- 후보가 READY로 바뀌면 주최자가 확정한다.
- 확정 화면에서 회의 제목과 날짜·시각을 확인한다.

## 4. 대표 시연 순서

```text
1. 생성
2. 요청 발송
3. 응답 대기
4. 응답 도착 상태로 진행
5. PENDING 결과
6. 수진에게 재요청
7. 수진 응답
8. 같은 후보 READY 재판정
9. 확정
```

DEV 화면 전환의 `current` 모드는 회의 상태와 선택 후보를 보존한다. 화면 이동 때문에 fixture를 다시 만들지 않는다.

평가자의 핵심 시연은 DEV 화면 전환에 의존하지 않는다. 응답 대기 화면의 `응답 도착 상태로 진행`이 검증된 PENDING fixture를 현재 회의 맥락에 결합하고 결과 보드로 연결한다. 이 전환은 배포된 프로토타입에서도 `프로토타입 시연`으로 명시한다.

## 5. P0 보류

- 계정 홈, 내 회의, 받은 요청, 알림
- 여러 회의 목록과 계정 수준 재진입
- 실제 계정 인증과 정교한 전달 상태
- 캘린더 연결 설정·권한·실제 API
- 공유 후 새 시간 범위 추가
- 장소, 회의실, 화상회의 링크
- 복잡한 마감 정책
- 핵심 흐름 밖 주변 화면의 모바일 최적화
- 최종 제품명

계정 화면의 구현 코드는 후속 제품 탐색 자료로 보존하되 DEV 화면에서만 접근한다. 제출용 기본 진입과 전역 내비게이션에는 노출하지 않는다.

## 6. 라우트와 셸 계약

- 기본 URL과 알 수 없는 URL은 `#/create`로 보낸다.
- 제출용 주최자 화면은 브랜드, `회의 시간 결정`, `처음부터`만 있는 집중 헤더를 사용한다.
- 참여자 토큰은 실제 계정 모델이 아니라 DEV 페르소나 전환 수단이다.
- 계정 라우트는 개발 환경에서만 접근할 수 있다.
- 생성과 결과는 같은 전역 프레임과 정렬축을 유지한다.

## 7. 완료 기준

- 도메인 테스트가 READY·PENDING·IMPOSSIBLE과 응답 전후 재판정을 보증한다.
- PENDING 시연은 수진의 응답 조건을 가진 후보를 선택한 상태로 시작한다.
- 결과 보드 상단 결론은 현재 선택 후보의 상태, 이유와 행동을 표시한다.
- 수진의 제출 뒤 현재 회의와 후보 선택을 유지한 결과 보드가 다시 계산된다.
- READY 후보를 확정하고 제목과 확정 시각을 확인할 수 있다.
- 핵심 화면에서 계정 메뉴와 보류 기능이 평가자의 주의를 분산시키지 않는다.
- PC와 모바일에서 핵심 경로의 조작과 정보 위계가 유지된다.
- 응답 대기 중 재알림은 화면 전환이나 레이아웃 변화 없이 완료 피드백을 주고, 해당 행에 실행 상태를 남긴다.
