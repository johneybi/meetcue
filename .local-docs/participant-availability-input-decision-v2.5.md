## 1. 결정 요약

참여자가 열린 모든 30분 칸에서 세 상태 중 하나를 반복해서 고르는 구조를 기본 경험으로 사용하지 않는다.

> 참여자는 먼저 열린 기간에 대한 자신의 일반적인 가용 패턴을 한 번 선택한다. 제품이 모든 칸에 기본 상태를 채우고, 참여자는 상태 브러시와 날짜 전체 적용을 사용해 예외만 수정한다.

새 응답에서 제공하는 기본 패턴은 다음 두 가지다.

- `대체로 가능해요`: 모든 열린 칸을 `AVAILABLE`로 채우고 기본 브러시를 `UNAVAILABLE`로 둔다.
- `가능한 시간이 적어요`: 모든 열린 칸을 `UNAVAILABLE`로 채우고 기본 브러시를 `AVAILABLE`로 둔다.

기본 상태 선택은 조용한 추론이 아니라 사용자의 명시적 선언이다. 제품은 선택 전에는 어떤 시간도 가능하거나 어렵다고 판정하지 않는다.

## 2. 문제

주최자 탐색 범위를 평일 업무시간처럼 넓게 제안하면 5일 기준 최대 90개의 30분 칸이 열릴 수 있다. 기존 참여자 화면은 각 칸마다 `가능`, `조정 가능`, `어려움` 버튼을 모두 반복했다.

이 구조는 다음 비용을 만든다.

- 같은 상태를 여러 칸에 반복 입력한다.
- 시간보다 반복되는 버튼이 더 강하게 보여 날짜와 연속성을 읽기 어렵다.
- 가능한 시간이 많은 사람과 적은 사람 중 어느 쪽에도 최적화되지 않는다.
- 주최자 입력을 줄인 비용이 참여자에게 이전된다.

## 4. 채택한 경험 계약

### 4.1 신규 응답

```text
회의 맥락과 조율 범위 확인
→ 일반적인 가용 패턴 선택
→ 제품이 전체 기본 상태 적용
→ 상태 브러시로 예외 수정
→ 응답 저장
```

- 패턴을 선택하기 전에는 시간 목록을 주 행동으로 노출하지 않는다.
- 패턴 선택 직후 모든 열린 칸은 명시적인 상태를 가진다.
- 사용자는 `처음부터`로 패턴 선택을 다시 할 수 있다.

### 4.2 상태 브러시

브러시는 다음 세 상태를 가진다.

- `가능해요`: 현재 일정 그대로 참석할 수 있다.
- `일정 조정해요`: 정해지면 다른 일정을 옮겨 참석한다.
- `어려워요`: 해당 시간에는 참석하기 어렵다.

사용자는 브러시를 먼저 고르고 시간 칸을 눌러 상태를 바꾼다. 각 시간 칸에 세 버튼을 반복하지 않는다. 날짜 헤더의 `이 날 전체 적용`으로 현재 브러시를 해당 날짜 전체에 적용할 수 있다.

### 4.3 기존 응답 수정

- 이미 제출한 응답은 패턴을 다시 묻지 않고 현재 상태를 그대로 연다.
- 새 초안을 저장하기 전까지 마지막 제출본이 주최자 판정에 유지된다.
- `가능하면 피하고 싶어요`는 `AVAILABLE` 또는 `ADJUSTMENT_COMMIT` 칸에서만 유지한다.

## 5. 미선택 상태와 제출 경계

- 신규 응답은 패턴 선택 뒤 모든 칸을 채우므로 일반적으로 `UNSET`이 남지 않는다.
- 이전 데이터 또는 이후 새로 확장된 범위에는 `UNSET`이 존재할 수 있다.
- `UNSET`이 남으면 제출 직전에만 어려움으로 저장된다는 사실을 확인한다.
- 초안은 제출 전까지 주최자 판정에 들어가지 않는다.

## 6. 역할 간 비용 배분

```text
제품: 주최자 기본 탐색 범위 생성
주최자: 자신의 예외만 제외·추가
제품: 참여자의 기본 상태 일괄 적용
참여자: 자신의 예외만 상태 변경
제품: 연속 가용성을 후보와 결정 상태로 해석
주최자: 최종 시간 결정
```

주최자와 참여자 모두 빈 시간표를 처음부터 제작하지 않는다. 제품은 기본값을 임의로 확정하지 않고 각 역할의 명시적 확인을 받는다.

## 7. 알려진 위험과 검증 질문

### 일반 패턴의 과도한 단순화

사용자의 일정이 날짜마다 크게 다르면 두 패턴 모두 정확하지 않을 수 있다. 패턴은 최종 응답이 아니라 편집 시작점임을 화면에서 이해할 수 있어야 한다.

### 기본값을 확인하지 않고 제출

전체 상태가 즉시 채워지므로 사용자가 예외를 검토하지 않을 수 있다. 다음을 검증한다.

- 패턴 선택 뒤 바뀐 전체 상태를 인지하는가?
- 다른 시간은 브러시로 수정해야 한다는 사실을 발견하는가?
- 제출 전에 자신의 응답을 빠르게 훑어볼 수 있는가?

### 모바일 연속 입력

P0은 기본 상태, 상태 브러시, 날짜 전체 적용을 우선 구현한다. 터치 드래그 연속 칠하기는 오조작과 스크롤 충돌을 함께 검증한 뒤 추가한다.

## 8. 구현 영향

- 참여자 신규 응답에 기본 패턴 선택 상태를 추가한다.
- 기본 패턴 선택 시 열린 모든 30분 슬롯을 한 번에 채운다.
- 반복되는 3버튼을 상단 상태 브러시 하나로 이동한다.
- 각 시간 행은 현재 상태와 시간만 보여주고, 누르면 선택한 브러시 상태로 바뀐다.
- 날짜 전체 적용과 처음부터 다시 선택을 제공한다.
- 참여자 응답 파생, 후보 판정과 제출 경계는 유지한다.
